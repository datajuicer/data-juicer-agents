data_juicer_agents.tools.retrieve.retrieve_operators.backend.result_builder module#

Shared helpers for building retrieval results and trace entries.

Extracted from backend.py and logic.py to eliminate duplicated code across the four retrieval backends (LLM, Vector, BM25, Regex).

data_juicer_agents.tools.retrieve.retrieve_operators.backend.result_builder.build_retrieval_item(tool_name: str, description: str = '', relevance_score: float = 0.0, score_source: str = '', operator_type: str = '', key_match: list[str] | None = None) → dict[str, Any][source]#

Build a standardised retrieval result item dict.

All string fields are stripped; relevance_score is cast to float. This is the single authoritative constructor for retrieval items used by every backend, replacing the ad-hoc dict literals that were previously duplicated across retrieve_ops_lm_items, retrieve_ops_bm25_items, and retrieve_ops_regex_items.

data_juicer_agents.tools.retrieve.retrieve_operators.backend.result_builder.filter_by_op_type(info_list: list[dict[str, Any]], op_type: str | None, type_key: str = 'class_type') → list[dict[str, Any]][source]#

Pre-filter an operator info list by op_type.

Falls back to the full list when the filter yields no results, so callers always receive a non-empty list as long as info_list itself is non-empty.

Parameters:

  • info_list – List of operator info dicts (e.g. from get_op_catalog()).

  • op_type – Operator type string to match (e.g. "filter"). When None or empty the full list is returned unchanged.

  • type_key – Dict key used to read the operator type from each entry.

data_juicer_agents.tools.retrieve.retrieve_operators.backend.result_builder.filter_by_tags(info_list: list[dict[str, Any]], tags: list[str] | None, tags_key: str = 'class_tags') → list[dict[str, Any]][source]#

Pre-filter an operator info list by tags (match-all semantics).

An entry matches only if its tag set contains all of the requested tags. Falls back to the full list when the filter yields no results, consistent with filter_by_op_type().

Parameters:

  • info_list – List of operator info dicts (e.g. from get_op_catalog()).

  • tags – Tag strings to match. When None or empty the full list is returned unchanged.

  • tags_key – Dict key used to read the tag list from each entry.
